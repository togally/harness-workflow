# 风险识别规则

**触发时机：** before-task hook，每个任务开始前自动扫描

## 高风险关键词

| 类别 | 关键词 |
|------|--------|
| 数据破坏 | 删除数据、清空、drop、truncate、rm -rf |
| 生产环境 | production、prod、线上、发布、release |
| 大范围变更 | 全量替换、重构所有、删除所有、migrate |
| 权限/安全 | sudo、chmod 777、credentials、密钥、token |
| 不可逆 Git | force push、reset --hard、branch -D |

## 风险处理规则

```
识别到高风险关键词
    ↓
评估两个维度：可逆性 + 影响范围

可逆 + 范围小（<5个文件，可 git 回滚）
    → 记录到 session-memory，继续执行

不可逆 或 范围大
    → 暂停执行
    → 向用户说明：操作内容、风险点、建议替代方案
    → 等待用户明确确认后再继续

涉及生产/安全/权限
    → 强制停止
    → 必须人工授权，不得自行判断
```

## 快速判断原则
- 能 git 回滚的 → 低风险
- 影响范围只在 workflow/ 内的 → 中风险（说明后继续）
- 影响项目代码或生产的 → 高风险（必须确认）

## 4. 接收方指引

### 4.1 加载步骤
1. **读取本文件**：了解任务状态和交接内容
2. **恢复状态文件**：
   - 确保 `runtime.yaml` 中的 `current_requirement`、`stage` 正确
   - 如有必要，更新 `locked_requirement`、`locked_stage`
3. **加载角色文件**：根据 `stage` 加载对应的角色文件
4. **加载经验文件**：根据 `stage` 加载相关经验文件
5. **加载变更文档**：读取 `change.md`、`plan.md`、`design.md`
6. **恢复会话记忆**：读取 `session-memory.md`，了解执行历史

### 4.2 继续工作的步骤
1. **验证交接完整性**：
   - 检查所有必须传递的文件是否存在且可读
   - 验证任务状态描述是否准确
   - 确认关键决策记录完整
2. **从交接点继续**：
   - 根据 `待完成的工作` 列表继续执行
   - 遵循 `plan.md` 中的剩余步骤
   - 如有疑问，参考 `关键决策记录`
3. **报告交接状态**：
   - 在 `session-memory.md` 中记录交接完成
   - 如有问题或疑问，及时记录

### 4.3 注意事项
- **未完成的任务项**：优先处理 `待完成的工作` 列表中的任务
- **已知风险**：[列出已知风险，如技术难点、依赖问题等]

# Change

## 1. Title

监控指标与阈值设计

## 2. Goal

定义上下文维护的监控指标和触发阈值，为后续维护动作提供决策依据：
- 指标：token 估算方法、消息条数、文件读取次数、会话时长等可量化指标
- 阈值：预警阈值、强制维护阈值、紧急阈值
- 指标采集方法：主 agent 定期估算、subagent 任务结束报告、用户手动检查点

## 3. Requirement

- `req-04-上下文维护机制设计`

## 4. Scope

**包含**：
- 定义监控指标清单及其量化方法：
  - Token 估算：基于消息历史长度、文件内容大小的估算公式
  - 消息条数：用户消息 + AI 回复的总条数
  - 文件读取次数：Read 工具调用次数
  - 会话时长：从会话开始到当前的时间
- 设计触发阈值：
  - 预警阈值（如 70-80% 最大上下文）：提醒用户上下文负载较高
  - 强制维护阈值（如 85-90%）：必须执行维护动作
  - 紧急阈值（>95%）：上下文已接近爆炸，需立即处理
- 定义指标采集方法：
  - 主 agent 定期估算：在关键节点（阶段转换、任务完成）估算 token
  - subagent 任务结束报告：每个 subagent 任务结束时报告消耗的上下文量
  - 用户手动检查点：用户可随时通过命令检查当前上下文负载

**不包含**：
- 维护动作的具体实现（属于 chg-02）
- handoff 交接协议设计（属于 chg-03）
- 工具层集成（属于 chg-04）
- 风险与恢复路径（属于 chg-05）
- 经验沉淀与角色职责更新（属于 chg-06）

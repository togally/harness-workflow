# Change: 三端支持

## 目标
为 harness install 命令添加交互式平台选择功能，支持 codex / qoder / cc 三端配置管理，包括备份、恢复和同步机制。

## 背景
用户需要在不同项目中灵活选择支持哪些 AI 平台，当前 install 命令会同时生成三个平台的配置，无法按需选择。

## 范围

**包含：**
- 创建统一顶层入口文件 `WORKFLOW.md`
- 三端入口文件（AGENTS.md / CLAUDE.md / .qoder/SKILL.md）简化并引用统一入口
- install 命令添加交互式平台勾选功能
- 备份机制：不支持的配置移动到 `.workflow/context/backup/`
- 恢复机制：重新勾选已移除平台时从备份恢复
- update 命令同步更新备份中的配置
- 备份目录结构设计

**不包含：**
- 各平台配置内容的修改
- 其他 harness 命令的变更

## 验收标准
1. 执行 `harness install` 时显示平台选择界面（多选框）
2. 勾选的平台生成对应配置，未勾选的配置移到备份目录
3. 再次执行 install 并勾选已移除平台时，从备份恢复（不重新生成）
4. 执行 `harness update` 时，备份中的配置同步更新
5. 备份目录结构清晰，可追溯

## 依赖
- 无前置依赖

## 风险
- 用户可能误删配置，需要提示备份位置
- 跨平台路径兼容性（Windows/macOS/Linux）

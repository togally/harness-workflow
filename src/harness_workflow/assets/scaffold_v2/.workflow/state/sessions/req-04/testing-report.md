**验证路径**：

- `chg-02-维护动作决策树/design.md`

**五项内容核查**：

1. **监控指标及其量化方法**：chg-01 定义了五项指标（Token 估算、消息条数、文件读取次数、会话时长、工具调用分布），每项均有明确量化方法和采集时机。✅

2. **触发阈值（至少两个级别）**：chg-01 定义三层阈值：
   - 预警阈值（黄色）：70-80% tokens，消息 >100，文件读取 >50，会话 >2h
   - 强制维护阈值（橙色）：85-90% tokens，消息 >150，文件读取 >80，会话 >4h
   - 紧急阈值（红色）：>95% tokens，消息 >200，文件读取 >100，会话 >6h
   满足"至少两个级别"的最低要求，实际提供三级。✅

3. **维护动作决策树**：chg-02 定义完整决策树，包含决策流程图、决策表（6种场景组合）、特殊情况处理（关键决策未记录、多指标同时触发、阶段转换时）。三种动作（`/compact`、`/clear`、新开 agent）均有适用场景、操作步骤、注意事项。✅

4. **`handoff.md` 交接协议模板**：chg-03 定义完整模板结构（7章节，含必填/可选字段说明）、交接内容规范、接收方指引规范（6步加载流程）及完整/简化版示例要求。✅

5. **各角色在上下文维护中的职责分工**：chg-06 明确三方职责：
   - 主 agent（WORKFLOW.md）：监控职责、触发职责、协调职责、交接管理
   - Subagent（六个角色文件）：消耗报告、清理建议、状态保存（executing 和 regression 额外说明）
   - 用户：关注提醒、确认高风险动作、提供反馈
   ✅

**结论**：五项内容均被完整覆盖，通过。

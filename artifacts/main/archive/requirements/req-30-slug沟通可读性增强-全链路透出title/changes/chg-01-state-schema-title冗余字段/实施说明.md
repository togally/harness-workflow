# 实施说明：chg-01 state schema — title 冗余字段

> 本文档首次提到 req-30（slug 沟通可读性增强：全链路透出 title）和 chg-01（state schema — title 冗余字段），后续沿用简写 id。

## 实际做了什么
- Step 1：`DEFAULT_STATE_RUNTIME` 新增 `current_requirement_title` / `current_regression_title` / `locked_requirement_title`（默认空串）；`save_requirement_runtime` 的 `ordered_keys` 按 id 字段相邻顺序插入三个 `*_title`，保证 yaml 字段顺序稳定。
- Step 2：新增 `_resolve_title_for_id(root, work_item_id)` helper（null-safe）：`req-*` 查 `state/requirements/`、`bugfix-*` 查 `state/bugfixes/`，其他前缀或缺文件返回空串。
- Step 3：改造所有写 `current_requirement` / `current_regression` / `locked_requirement` 的 runtime 写入点——`create_requirement` / `create_bugfix` / `harness_regression` / `regression_action` 的 4 处清空分支 / `archive_requirement` 切换 active / `enter_workflow` / `set_regression_mode` / `set_conversation_mode` 均同步维护对应 `*_title`（新建用 `_resolve_title_for_id`，清空直接写空串）。
- Step 4：`create_requirement` / `create_bugfix` 对空 title 抛 `SystemExit` 的现有校验保留，未新增破坏性行为。
- Step 5：活跃需求 `req-30` 在 `state/requirements/req-30-slug沟通可读性增强-全链路透出title.yaml` 中已有 `title`，验证非空。
- Step 6：新增 `tests/test_runtime_title_fields.py` 共 12 条用例覆盖 schema / helper / 同步写入 / 向后兼容。

## 未做与原因

- 未创建 `tests/test_runtime_title_fields.py` 之外的测试拆分文件（plan 明确允许合并或独立，独立更便于 grep）。

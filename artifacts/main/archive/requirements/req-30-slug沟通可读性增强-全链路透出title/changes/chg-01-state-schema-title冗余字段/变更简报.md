# 变更简报：chg-01 state schema — title 冗余字段

> 隶属 req-30（slug 沟通可读性增强：全链路透出 title），方案 B 的 L3 层落地（AC-05 / AC-06）。

## 变更名
- runtime.yaml / state yaml 层 schema 扩展：新增 `*_title` 冗余字段并由统一写入 helper 维护。

## 解决什么问题
- AC-05：`runtime.yaml` 仅存 id 无 title，subagent 需二次打开 `state/requirements/*.yaml` 才能拿到 title（场景 4 痛点）。
- AC-06：所有写 runtime id 的代码路径无"同步写 title"约束，易漂移。

## 怎么做
- `DEFAULT_STATE_RUNTIME` 新增 `current_requirement_title` / `current_regression_title` / `locked_requirement_title`，默认空串，位置紧邻对应 id。
- 新增 `_resolve_title_for_id(root, work_item_id)` helper：按 id 前缀分流到 state/requirements 或 state/bugfixes，null-safe 返回空串。
- 改造 10+ runtime id 写入点（create_requirement / create_bugfix / harness_regression / regression_action / archive_requirement / enter_workflow / set_regression_mode / set_conversation_mode），id 变更时同步刷新 `*_title`。
- 向后兼容：旧 runtime.yaml 缺 `*_title` 字段时，`load_requirement_runtime` 回填空串不炸。

## 影响范围
- `src/harness_workflow/workflow_helpers.py`：schema + helper + 多处写入点。
- `tests/test_runtime_title_fields.py`（新增）：12 条单元测试。

## 预期验证
- `pytest tests/test_runtime_title_fields.py` 12/12 绿。
- 新建 req 后 `cat .workflow/state/runtime.yaml` 能看到 `current_requirement_title` 非空。
- 全量 pytest 零回归（175 passed 基线）。

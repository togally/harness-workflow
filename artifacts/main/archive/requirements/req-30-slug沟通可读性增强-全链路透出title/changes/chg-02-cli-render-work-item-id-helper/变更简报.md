# 变更简报：chg-02 CLI 渲染 — render_work_item_id helper

> 隶属 req-30（slug 沟通可读性增强：全链路透出 title），方案 B 的 L2 层落地（AC-03 / AC-04 / AC-06 读取侧 / AC-09）。

## 变更名
- CLI stdout 渲染层统一 helper：`render_work_item_id` 把工作项 id 转显示字符串 `{id}（{title}）`，所有打印 id 的命令走此入口。

## 解决什么问题
- AC-03：`harness status` / `next` / `ff` / `suggest --list` 打印 id 时不带 title，读者需反查目录才能识别含义。
- AC-04：title 缺失时必须降级显示 `(no title)`，不能因缺字段 crash。
- AC-06 读取侧：统一"runtime 缓存优先、state 权威 fallback"的数据源策略。

## 怎么做
- 新增 `render_work_item_id(id, *, runtime, root)`：runtime 缓存 → `_resolve_title_for_id`（state）→ sug 文件 frontmatter / body 首行 → `{id} (no title)` 降级。
- 辅助 `_resolve_title_for_suggestion` 处理 sug 分支（legacy sug 无 frontmatter 时读 body 首行 40 字符）。
- 辅助 `_render_id_list` 批量渲染 active_requirements。
- 改造 `workflow_status` / `workflow_fast_forward` / `list_suggestions` 三条命令入口走 helper；`workflow_next` 主输出不涉及 id 无需改动。

## 影响范围
- `src/harness_workflow/workflow_helpers.py`：新增 3 helper + 改造 3 命令入口。
- `tests/test_render_work_item_id.py`（新增）：12 条单测（6 核心分支 + 3 sug + 3 workflow_status smoke）。
- 用户侧 stdout：`current_requirement` / `active_requirements` / `sug-NN` 等行均携带 title。

## 预期验证
- `pytest tests/test_render_work_item_id.py` 12/12 绿。
- `harness status` 手工 smoke 输出 `current_requirement: req-30（slug 沟通可读性增强：全链路透出 title）`。
- 全量 pytest 零回归。

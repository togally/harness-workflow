# 实施说明：chg-02 CLI 渲染 — render_work_item_id helper

> 承接 req-30（slug 沟通可读性增强：全链路透出 title），本次实施 chg-02（CLI 渲染 — render_work_item_id helper），后续沿用简写 id。

## 实际做了什么
- Step 1：新增 `render_work_item_id(work_item_id, *, runtime=None, root=None)` helper，按优先级 runtime 缓存 → state fallback → sug frontmatter/body → `(no title)` 降级；空 id 返回 `(none)`。
- Step 1（辅）：新增 `_resolve_title_for_suggestion`（读 sug 文件 frontmatter.title，失败降级 body 首行 40 字符截断，覆盖 legacy sug 无 frontmatter 场景）。
- Step 1（辅）：新增 `_render_id_list` 批量渲染 helper。
- Step 2：改造 `workflow_status`——`current_requirement` / `locked_requirement` / `current_regression` 三行同行附带 title；`active_requirements` 用批量 helper 渲染；原有 `requirement_title` 独立行保留（冗余不误导）。
- Step 3：`workflow_next` 本身打印不涉及 id（仅 `Workflow advanced to {stage}`），无需改动（`workflow_fast_forward` 分担 ff 场景 id 打印）。
- Step 4：改造 `workflow_fast_forward`——有 `operation_target` 时打印 `Workflow advanced to {stage} for {id}（{title}）`，空时保留原格式。
- Step 5：改造 `list_suggestions`——每行 sid 走 helper，打印 `sug-NN（title）[status] (...) body`。
- Step 6：新增 `tests/test_render_work_item_id.py` 共 12 条用例，覆盖核心分支 6 + sug 分支 3 + `workflow_status` 集成 smoke 3。

## 未做与原因
- 未改 `workflow_next` 主输出（`Workflow advanced to {stage}` 不涉及 id）；真正 id 打印集中在 ff 与 status，符合 plan。
- 未改 `record_feedback_event` payload（后端机器消费，plan 明确不改）。
- 未对 `reg-*` 渲染专门分支：若 runtime 缓存命中 `current_regression_title`（chg-01 已在 harness_regression 入口写入）即带 title，否则降级 `(no title)`；与 plan 一致。
- 未做全局缓存（plan R3 评估 O(n·m) < 30 文件可接受）。

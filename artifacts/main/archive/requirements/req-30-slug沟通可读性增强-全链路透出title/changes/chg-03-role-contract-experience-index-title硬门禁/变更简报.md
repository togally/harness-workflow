# 变更简报：chg-03 角色契约 — id + title 硬门禁

> 隶属 req-30（slug 沟通可读性增强：全链路透出 title），方案 B 的 L1 + L4 索引层落地（AC-01 / AC-02 / AC-08 / AC-10 自证）。

## 变更名
- 把"id + title 同时出现"从软约定升格为契约硬门禁：`stage-role.md` 契约 7 + 7 个 stage 角色样本 + director briefing 模板 + `experience/index.md` 校验规则。

## 解决什么问题
- AC-01：agent 对人 / 跨 agent 汇报中，工作项 id 首次出现未带 title。
- AC-02：stage 角色文件的"汇报示例"章节缺双字段模板。
- AC-08：`experience/index.md` 的"来源"列只写裸 id，新增条目无校验。

## 怎么做
- `stage-role.md` 新增 `### 契约 7：id + title 硬门禁`，含规则 / 校验方式 / fallback + "与契约 1-6 并列生效不覆盖"注脚（R3）。
- 7 个 stage 角色文件（requirement-review / planning / executing / testing / acceptance / regression / done）在模板章节各加 1 段契约 7 注脚 + 汇报样本；`done.md` 退出条件新增 grep 校验项。
- `technical-director.md` 的 Step 4 派发段落新增"Subagent briefing 模板"：`current_requirement_title` 双字段 + 正文形如 `{id}（{title}）`。
- `experience/index.md` 新增"来源字段校验规则"段 + 正反例 + 生效时机。
- 示范性回填 `experience/roles/planning.md` 经验一的来源段为 `req-29（批量建议合集 2 条）—` 粒度。

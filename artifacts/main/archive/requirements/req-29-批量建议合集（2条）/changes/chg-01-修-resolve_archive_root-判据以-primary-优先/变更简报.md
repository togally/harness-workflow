# 变更简报：chg-01 修 resolve_archive_root 判据以 primary 优先

## 变更名
- 把归档路径解析改成"primary 始终优先，legacy 仅在显式 opt-in 时使用"，让新归档不再因老仓 legacy 非空而回落到旧目录。

## 解决什么问题
- 关联 **AC-03**：`resolve_archive_root` 当前在 legacy 非空时会自动降级返回 legacy，导致新归档写到 `.workflow/flow/archive/` 而不是 `artifacts/{branch}/archive/`，破坏了 req-27 建立的分支目录契约。

## 怎么做
- 调整 `resolve_archive_root` 判据链：primary 始终优先；legacy 非空时仅在 stderr 打一次迁移提示，不再默认降级。
- 新增显式 opt-in 机制（推荐 `prefer_legacy: bool` 参数 或 `HARNESS_ARCHIVE_LEGACY=1` 环境变量，executing 阶段敲定其一）。
- 扩展 `tests/test_archive_path.py`：新增 "legacy 非空时返回 primary" 和 "opt-in 时返回 legacy" 两条用例。
- 扫描所有 `resolve_archive_root(` 调用点，确认无人依赖旧降级行为。

## 影响范围
- `src/harness_workflow/workflow_helpers.py`（`resolve_archive_root`，约 4280 行）
- `tests/test_archive_path.py`
- 间接影响：所有调用 `harness status` / `harness archive` 时的 stderr 输出（新增迁移提示）。

## 预期验证
- `pytest tests/test_archive_path.py -v` 通过，且新增两条用例命中。
- 在含 legacy 非空归档的仓库执行 `harness status`：stderr 出现 `run harness migrate archive` 提示，但新归档落到 `artifacts/{branch}/archive/`。
- `grep -rn "resolve_archive_root" src` 的调用方全部兼容新签名（默认参数不破坏老调用）。
- 函数改动 diff 仅限本函数体 + 必要签名扩展，未牵扯 `resolve_requirement_root` / `resolve_bugfix_root`。

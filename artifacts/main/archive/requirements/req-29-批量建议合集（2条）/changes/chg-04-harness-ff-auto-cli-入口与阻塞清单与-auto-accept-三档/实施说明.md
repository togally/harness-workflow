# 实施说明：chg-04 harness ff --auto CLI 入口与阻塞清单与 auto-accept 三档

## 实际做了什么
- Step 1：`cli.py` 的 `ff` 子命令新增 `--auto` / `--auto-accept {low,all}` 两个 flag；主分派中校验 `--auto-accept` 必须配合 `--auto`，否则 argparse 报错退出。
- Step 2：新建 `src/harness_workflow/ff_auto.py`，实现 `workflow_ff_auto(root, auto_accept, input_reader, output_writer)` 主入口：按 `operation_type` 选 sequence，把 stage 推进到 `acceptance` 前一步停下。
- Step 3：复用 chg-03 的 `read_decision_log` + `is_blocking_decision` 做三档 ack；阻塞类别的决策点**强制交互**（即使 `--auto-accept all` 也不自动通过）。
- Step 4：进入 acceptance 前调用 `write_decision_summary` 把 `决策汇总.md` 落到 `artifacts/{branch}/requirements/{req-id}-{slug}/`。
- Step 5：新增 `tests/test_ff_auto.py` 7 条用例；整仓 `python3 -m unittest discover tests` 159 → 166 全过。

## 未做与原因
- **不在本仓真跑 `harness ff --auto`**：会污染 runtime.yaml 和 decision-log，真实 smoke 留给 chg-05。
- **不把阻塞检测 hook 进 stage 内部 action 流**：stage 内部没有统一的 action 拦截点，按变更简报约定改为暴露 `check_blocking_before_action` 供 subagent 产出决策点前自检。

## 关键文件变更
- `src/harness_workflow/cli.py`：ff 子命令扩 flag + 主分派路由（+14 行）。
- `src/harness_workflow/ff_auto.py`（新增）：`workflow_ff_auto` / `check_blocking_before_action` / `_should_auto_ack` / `_advance_to_stage_before_acceptance`。
- `tests/test_ff_auto.py`（新增）：7 条用例覆盖 stage 推进、三档 ack、汇总落盘、helper 矩阵。

## 已知限制
- `_resolve_branch` 仅从 git + 回退 `main`，未读 `.workflow/config`；若项目在非主分支工作需要后续扩展。
- 本 change 未强制把所有决策点"写日志"流程接到 CLI，依赖上层 subagent 自觉调 `append_decision`；acceptance 前 CLI 只做 ack + 汇总。

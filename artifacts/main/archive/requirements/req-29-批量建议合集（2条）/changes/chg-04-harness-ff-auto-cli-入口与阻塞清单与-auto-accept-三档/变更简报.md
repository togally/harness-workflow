# 变更简报：chg-04 harness ff --auto CLI 入口与阻塞清单与 auto-accept 三档

## 变更名
- 把 `--auto` 和 `--auto-accept {low,all}` 两个 flag 装进 `harness ff`，实现自主推进主循环、5.1 阻塞清单检测、5.3 三档语义、5.2 双轨日志 + 汇总落盘。

## 解决什么问题
- 直接对应 **AC-01**（`ff --auto` 主路径）以及 5.1 / 5.2 / 5.3 三份决策规约。chg-03 已经做好数据层契约，chg-04 接上 CLI 入口与交互，让用户真的能跑起来。

## 怎么做
- CLI 层扩 `harness ff` 增加两个 flag；`--auto-accept` 必须依赖 `--auto`。
- 抽出 `auto_runner.py` 主循环：按 stage 顺序推进，每个决策先 `detect_blocking` 再 `append_decision`；命中阻塞立即 abort，不写日志。
- 实现 `auto_blocking.py`：`BLOCKING_CATEGORIES` 按 5.1 清单硬编码 8 条规则（rm-rf、force push、跨 req、archive 删、凭证、敏感配置、依赖变更、schema）。
- 进入 acceptance 前：先落盘 `决策汇总.md`，再按三档交互（None 全交互 / low 仅 low 自动 / all 全自动）。
- 新增 `tests/test_ff_auto.py` 7 个用例覆盖三档、阻塞、汇总顺序、CLI 校验。

## 影响范围
- 修改：`src/harness_workflow/tools/harness_ff.py`、`src/harness_workflow/workflow_helpers.py` 或新文件 `src/harness_workflow/auto_runner.py`
- 新增：`src/harness_workflow/auto_blocking.py`、`tests/test_ff_auto.py`
- 同步：`.workflow/context/roles/harness-manager.md`（若该文件描述 `harness ff` 入口）
- 本 change **不跑 `ff --auto`**，真实 smoke 由 chg-05 负责。

## 预期验证
- `pytest tests/test_ff_auto.py -v` 全绿（7 条用例）；`tests/test_auto_decisions.py` / `tests/test_archive_path.py` 无回归。
- `harness ff --help` 输出包含 `--auto` / `--auto-accept` 的中文帮助。
- 单测断言 `决策汇总.md` 落盘顺序先于交互提示。
- 用例 `test_auto_accept_requires_auto` 证明仅传 `--auto-accept low` 会报错退出。

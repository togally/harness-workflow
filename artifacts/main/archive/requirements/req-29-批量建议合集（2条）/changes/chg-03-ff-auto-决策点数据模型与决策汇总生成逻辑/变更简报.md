# 变更简报：chg-03 ff auto 决策点数据模型与决策汇总生成逻辑

## 变更名
- 给 `--auto` 模式打好"数据层地基"：一个 `DecisionPoint` 数据类、一套 `decisions-log.md` 读写 helper、一个 `决策汇总.md` 渲染函数，并把"决策汇总.md"加进 stage-role 的对人文档契约。

## 解决什么问题
- 关联 **AC-02**（决策点数据契约）、并为 **AC-01** 的"acceptance 前汇总"打底。没有稳定的数据契约，chg-04 的 CLI 入口没法落地；没有 stage-role 契约行，主 agent 不知道该产出这份对人文档。

## 怎么做
- 新建 `src/harness_workflow/auto_decisions.py`，内含 `DecisionPoint` dataclass（id / ts / stage / risk / options / choice / reason / extra）。
- 实现 `append_decision` + `load_decisions`：把日志写到 `.workflow/flow/requirements/{req-id}/decisions-log.md`，按 fenced YAML block 追加，支持幂等读取。
- 实现 `render_decision_summary`：纯函数，按 high/medium/low 分组渲染 Markdown（供 chg-04 写到 `artifacts/{branch}/requirements/{req-id}-{slug}/决策汇总.md`）。
- 修改 `.workflow/context/roles/stage-role.md`：契约 3 表格加一行"决策汇总.md"，契约 4 加一条"--auto 激活时必须产出"的硬门禁。
- 新增 `tests/test_auto_decisions.py` 6 个用例。

## 影响范围
- 新增：`src/harness_workflow/auto_decisions.py`、`tests/test_auto_decisions.py`
- 修改：`.workflow/context/roles/stage-role.md`（只动对人文档契约的两处）
- 无 CLI 入口改动（由 chg-04 接管）；无现有 helper 改签名。

## 预期验证
- `pytest tests/test_auto_decisions.py -v` 全绿（6 条用例）。
- Python REPL 跑一次 `append_decision` + `render_decision_summary`，人眼检查分组输出与风险标记。
- `git diff stage-role.md` 只命中契约 3 表格新增一行 + 契约 4 新增一段。
- `grep -rn "DecisionPoint" src` 仅本模块；未越界改 `harness_ff.py`。

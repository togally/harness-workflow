# 实施说明：chg-03 ff auto 决策点数据模型与决策汇总生成逻辑

## 实际做了什么
- 新建 `src/harness_workflow/decision_log.py`（394 行），导出 `DecisionPoint` / `append_decision` / `read_decision_log` / `render_decision_summary` / `write_decision_summary` / `BLOCKING_CATEGORIES` / `is_blocking_decision`，覆盖 5.2 双轨的 flow 侧日志读写与 artifacts 侧汇总产出。
- 日志用 fenced YAML block 追加到 `.workflow/flow/requirements/{req-id}/decisions-log.md`，ID `dec-NNN` 自动单调递增；`decisions-log.md` 不存在自动建目录与表头。
- `render_decision_summary` 为纯函数，按 high → medium → low 三档分组，每条输出时间/stage/描述/可选项/选择/理由；`write_decision_summary` 负责落盘到 `artifacts/{branch}/requirements/{req-id}-{slug}/决策汇总.md`，slug 从现有目录推断，找不到时兜底 `{req-id}` 目录。
- `is_blocking_decision` 对 requirement.md §5.1 的 8 条阻塞类别做 keyword 命中，供 chg-04 CLI 入口调用。
- `.workflow/context/roles/stage-role.md` 与 scaffold 同名文件"契约 3"表格后各追加 1 条"决策汇总.md" 契约条目，diff=0 同步。
- 新增 `tests/test_decision_log.py`（218 行，14 个用例）。

## 未做与原因
- 未扩展 `harness ff` CLI、`--auto-accept` 三档交互、smoke 脚本——chg-04 / chg-05 范围。

- `src/harness_workflow/decision_log.py`：新模块（数据契约 + 日志读写 + 汇总渲染 + 阻塞检测）。
- `tests/test_decision_log.py`：append / roundtrip / 分组渲染 / 写盘路径 / 阻塞检测共 14 条用例。
- `.workflow/context/roles/stage-role.md` 与 `src/harness_workflow/assets/scaffold_v2/.workflow/context/roles/stage-role.md`：对人文档契约追加"决策汇总.md"行。

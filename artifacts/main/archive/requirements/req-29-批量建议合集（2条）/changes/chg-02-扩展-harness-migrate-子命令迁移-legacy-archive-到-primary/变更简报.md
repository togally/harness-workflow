# 变更简报：chg-02 扩展 harness migrate 子命令迁移 legacy archive 到 primary

## 变更名
- 给 `harness migrate` 增加 `archive` 资源，一键把老仓 `.workflow/flow/archive/` 下的历史归档迁到 `artifacts/{branch}/archive/`，默认 copy-verify-delete 幂等可回滚。

## 解决什么问题
- 关联 **AC-04**：chg-01 修好了判据后，仓库里还堆着 req-26/27/28、bugfix-3/4/5/6 等 legacy 归档——需要一个命令把它们收敛到规范路径，否则 `harness status` 仍会一直告警。

## 怎么做
- 新增 `migrate_archive` helper：扫描 legacy 子条目，按 `<branch>/...` 或裸条目分别映射到 `artifacts/<branch>/archive/...` 或 `artifacts/main/archive/<name>`。
- copy-then-verify-then-unlink：先 copy、再按 sha256 比对文件列表、最后删 legacy；任意步骤失败就回滚 + 保留 legacy 完整。
- 幂等：目标已存在且内容一致 → 跳过；不一致 → 进冲突列表、退出码非零、一个文件都不动。
- CLI：`harness migrate archive [--dry-run]`，语义与现有 `harness migrate requirements` 对齐。
- 新增 `tests/test_migrate_archive.py` 覆盖 happy path、幂等、冲突、dry-run、multi-branch 五个场景。

## 影响范围
- `src/harness_workflow/tools/harness_migrate.py`（CLI 扩展）
- `src/harness_workflow/workflow_helpers.py`（新增 `migrate_archive` helper）
- `tests/test_migrate_archive.py`（新建）
- 运行时影响：执行迁移后 `.workflow/flow/archive/` 清空，`artifacts/{branch}/archive/` 出现迁移产物。

## 预期验证
- `pytest tests/test_migrate_archive.py -v` 五个用例全绿。
- 本仓 `harness migrate archive --dry-run` 输出包含 req-26 / req-27 / req-28 / bugfix-3/4/5/6 等预期条目。
- 实际迁移后 `harness status` / `harness archive` 不再出现 `using legacy archive path` 警告。
- 重复跑迁移命令 → 退出码 0、打印 `already migrated`，没有副作用。

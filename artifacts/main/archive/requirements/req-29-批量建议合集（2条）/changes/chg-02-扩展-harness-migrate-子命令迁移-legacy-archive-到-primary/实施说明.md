## 已知限制
- `shutil.move` 跨设备会自动 copy+unlink，但非原子；若中断可能留"两边都有"的脏状态，与 `migrate_requirements` 同风险级别，依赖人工 reconcile（`conflict` 分支会打印源/目标路径）。
- 带分支前缀的 legacy 条目会被"重新归位"到当前 git 分支：若 legacy 下是 `main/...`、当前 git branch 不是 `main`，迁移结果会落到当前分支而非原分支；实施按 briefing 的"branch 由当前仓库解析"语义执行。

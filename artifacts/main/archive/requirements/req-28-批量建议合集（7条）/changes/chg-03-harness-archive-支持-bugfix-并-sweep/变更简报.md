# 变更简报：chg-03 harness archive 支持 bugfix + sweep

## 变更名

- 让 `harness archive` 识别 bugfix 目录并分流到 `artifacts/{branch}/archive/bugfixes/`，并在本 change 内一次性把 bugfix-3/4/5/6 全部归档干净。

## 解决什么问题

- 对应 AC-14：当前 `harness archive` 只扫 `requirements/`，bugfix 归不了档；`active_requirements` 长期挂着 4 条历史 bugfix，状态混乱。

## 怎么做

- `list_done_requirements` 扫 `requirements/` + `bugfixes/` 两处。
- `archive_requirement` 按 id 前缀分流落地根目录。
- 新增 `scripts/sweep_active_bugfixes.py`（或 `--sweep-bugfixes` 子选项），默认 `--dry-run`，`--apply` 才写。
- 对非 done 状态的 bugfix-3/4/5 以 `--force-done` 走完；bugfix-6 依赖 chg-02 回填后正常归档。

## 影响范围

- 代码：`src/harness_workflow/workflow_helpers.py`、`scripts/sweep_active_bugfixes.py`（新文件）。
- 用户路径：`harness archive`、`harness status`。
- 归档产物：`artifacts/main/archive/bugfixes/bugfix-{3,4,5,6}-*/`。

## 预期验证

- sweep 执行后 `harness status` 的 `active_requirements` 只剩 `req-28`。
- `ls artifacts/main/archive/bugfixes/` 可见 bugfix-3/4/5/6 四个归档目录。
- 新建 req + 归档的回归测试不受本 change 影响。
- `tests/test_archive_bugfix.py`（或扩展用例）全绿。

# 实施说明：chg-03 harness archive 支持 bugfix + sweep

## 实际做了什么

- Step 1：扩展 `list_done_requirements`（`workflow_helpers.py:4144`）同时扫 `.workflow/state/requirements/` 与 `.workflow/state/bugfixes/`，每条带 `kind: req/bugfix` 标签。
- Step 2：重写 `archive_requirement`（`workflow_helpers.py:4419`）按 id 前缀分流：`req-*` → `archive/requirements/<dir>`、`bugfix-*` → `archive/bugfixes/<dir>`；bugfix 分支从 `.workflow/state/bugfixes/` 取 state yaml，不再误触 sessions 迁移与 artifact 生成；`current_requirement` 仅当精确命中被归档 id 时才清空，保护主 req-28 不被 sweep 切走。
- Step 3：新增 `resolve_bugfix_root`（`workflow_helpers.py:4304`）与 `--force-done` 开关。CLI 层（`cli.py:198`、`cli.py:372` + `tools/harness_archive.py`）新增 `--force-done`，对 bugfix 绕过 stage=='done' 检查、强置 yaml 为 done/当前时间后再归档；对 `req-*` 是 no-op。
- Step 5：新增 `tests/test_archive_bugfix.py`（4 条用例），同步更新 `tests/test_archive_path.py` 与 `tests/test_smoke_req26.py` 对新路径 `archive/requirements/` 的断言。
- 设计决策：选择 `--force-done` 开关而非 `--sweep-bugfixes` 子命令——实现面更小、CLI 形态与现有 `archive` 一致，sweep 由主 agent 对 4 个 id 循环执行即可。

## 未做与原因

- Step 4（实际执行 sweep bugfix-3/4/5/6）：briefing 硬约束"不跑真 `harness archive --force-done`"，交主 agent 执行。

## 关键文件变更

- `src/harness_workflow/workflow_helpers.py`：`list_done_requirements` 扩扫、`archive_requirement` 重写分流、新增 `resolve_bugfix_root`。
- `src/harness_workflow/tools/harness_archive.py`：CLI 入口新增 `--force-done`。
- `src/harness_workflow/cli.py`：argparse 加 `--force-done`；archive 候选扫描扩至 bugfixes；`--force-done + id` 跳过交互直发 helper。
- `tests/test_archive_bugfix.py`（新增 4 用例）；`tests/test_archive_path.py` / `tests/test_smoke_req26.py` 路径断言随新规对齐。

## 已知限制

- bugfix 归档不走 `generate_requirement_artifact`（缺少 `requirement.md` 结构），也不迁 `state/sessions/`；如后续要给 bugfix 单独产一份制品摘要，需再起一轮。
- `--force-done` 无 PASS 校验（briefing 明确采用简化方案，由操作者负责）。

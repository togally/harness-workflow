# 变更简报：chg-01 修复 harness suggest CLI

## 变更名

- 给 `harness suggest` 系列命令补上 filename fallback、跨 archive 的编号单调递增，并把"新 sug 必带 frontmatter"升级为硬门禁。

## 解决什么问题

- 对应 AC-15：当前 sug 文件若缺 frontmatter，`--apply` / `--delete` 永远失败；且 `create_suggestion` 清空当前目录后编号会从 01 重新开始，与历史归档冲突。

## 怎么做

- `apply_suggestion` / `delete_suggestion` / `archive_suggestion` 增加 filename 正则 fallback（`^sug-(\d+)`）。
- `create_suggestion` 同时扫 `suggestions/` 与 `suggestions/archive/` 取最大编号 +1。
- `done.md` / `stage-role.md` + scaffold_v2 同步追加"新 sug 必带 frontmatter"硬门禁。
- `create_suggestion` 写入前校验 frontmatter 必填字段，缺失即拒。

## 影响范围

- 代码：`src/harness_workflow/workflow_helpers.py`、`src/harness_workflow/scaffold_v2/roles/done.md`、`scaffold_v2/roles/stage-role.md`。
- 规范：`.workflow/context/roles/done.md`、`.workflow/context/roles/stage-role.md`。
- 用户路径：`harness suggest --apply/--delete/--archive`、`harness suggest`（新建）。

## 预期验证

- 无 frontmatter 的 sug 文件也能被 `--delete` / `--apply` 成功处理。
- 清空 `suggestions/` 后新建 sug，编号不与 `archive/` 已有 sug 冲突。
- 直接调用写入新 sug 时缺 frontmatter → 报错退出。
- `tests/test_suggest_cli.py` 3 条用例全部通过。
- done.md / stage-role.md 可 grep 到"新 sug 必带 frontmatter"条目。

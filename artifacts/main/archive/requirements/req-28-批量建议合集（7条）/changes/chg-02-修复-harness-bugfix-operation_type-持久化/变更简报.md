# 变更简报：chg-02 修复 harness bugfix 闭环

## 变更名

- 让 `harness bugfix` 能把 `operation_type`/`operation_target` 写入并持久化到 `runtime.yaml`，并在 stage 推进时同步到 bugfix yaml。

## 解决什么问题

- 对应 AC-12：当前 `harness bugfix` 创建 bugfix 后 runtime.yaml 缺关键字段，`harness next` 从 regression 推不动，bugfix 全链路断掉。

## 怎么做

- `create_bugfix` 末尾写 `operation_type = "bugfix"` + `operation_target = "<id>"` 到 runtime.yaml。
- `save_requirement_runtime` / `load_requirement_runtime` 保留这两个字段，round-trip 不被裁剪。
- `harness next` 检测到 `operation_type == "bugfix"` 时，把 new stage 同步写到 `.workflow/state/bugfixes/<id>.yaml`（AC-03 对 bugfix 生效）。
- 提供一次性回填脚本 `scripts/backfill_bugfix_runtime.py`。

## 影响范围

- 代码：`src/harness_workflow/workflow_helpers.py`、`scripts/backfill_bugfix_runtime.py`（新文件）。
- 用户路径：`harness bugfix`、`harness next`、`harness status`。
- 状态文件：`.workflow/state/runtime.yaml`、`.workflow/state/bugfixes/*.yaml`。

## 预期验证

- `harness bugfix "x"` 后 runtime.yaml 含 `operation_type: bugfix` + `operation_target: bugfix-NN`。
- 任意一次 `harness next` / `harness status` 后两字段不丢。
- stage 推进后，`.workflow/state/bugfixes/bugfix-NN.yaml` 的 stage 与 runtime 一致。
- `tests/test_bugfix_runtime.py` 3 条用例全部通过。
- 回填脚本 `--dry-run` 能识别历史缺字段 bugfix。

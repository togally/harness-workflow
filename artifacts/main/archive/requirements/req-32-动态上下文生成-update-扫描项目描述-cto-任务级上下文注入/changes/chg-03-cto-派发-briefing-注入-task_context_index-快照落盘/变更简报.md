# 变更简报：chg-03（CTO 派发 briefing 注入 task_context_index + 快照落盘）

## 变更名

- 让主 agent 派发 subagent 时按任务内容产出 ≤ 8 条上下文索引，同步落快照到状态仓，补齐 req-32（动态上下文生成：update 扫描项目描述 + CTO 任务级上下文注入）Part B 闭环。

## 解决什么问题

- 覆盖 req-32 的 AC-03 / AC-04 / AC-07：subagent 不再盲扫全量 `.workflow/context/index.md`，按索引最小加载、未命中静默回退；索引同时落快照供事后复查与优化迭代。

## 怎么做

- 扩展 `_build_subagent_briefing`：新增 `task_context_index`（list，每条 `{path, reason}`）与 `task_context_index_file`（快照相对路径）双字段。
- 新增 `_build_task_context_index`：读 `.workflow/context/project-profile.md` 做领域加权，缺失时退化为 stage 默认推荐集；硬上限 8 条，超限截断 + stderr `warn: task_context_index truncated from N to 8`。
- 新增 `_write_task_context_snapshot`：落盘到 `.workflow/state/sessions/{req-id}/task-context/{stage}-{seq}.md`，frontmatter 含 `req_id` / `stage` / `ts` / `index_count`，{seq} 按目录扫描递增。
- 覆盖 `harness next --execute` / `harness ff --auto` / regression 三条派发路径；subagent C2 回退语义通过文档 + `_resolve_task_context_paths` 单测覆盖，CLI 不强制。
- 归档沿用既有 `harness archive`（session 目录随 req 迁移），不写新迁移代码。

## 影响范围

- 修改：`src/harness_workflow/workflow_helpers.py`、`.workflow/context/roles/stage-role.md` 及其 `scaffold_v2/` 镜像。
- 新增：`tests/test_task_context_index.py`、`tests/test_task_context_snapshot.py`。
- 用户侧：`.workflow/state/sessions/{req-id}/task-context/*.md` 首次出现；briefing JSON 多两个字段。

## 预期验证

- 本仓 `harness next --execute` stdout briefing 含 `task_context_index` 与 `task_context_index_file` 字段。
- 执行后 `.workflow/state/sessions/req-32/task-context/executing-01.md` 存在，frontmatter 四字段齐全，正文每行 `{path}: {reason}`。
- 连续派发同 stage 产生 `-02.md` / `-03.md` 递增。
- `pytest -q` 零回归；`harness validate --contract all` 绿。

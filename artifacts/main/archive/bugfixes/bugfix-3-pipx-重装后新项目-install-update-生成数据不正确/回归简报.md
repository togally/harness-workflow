# 回归简报：bugfix-3 pipx 重装后新项目 install/update 生成数据不正确

## 问题
pipx 重装后，在目标项目执行 `harness install` + `harness update`，用户感知"生成数据不对"；实测是新模板章节未下发，且 `experience/index.md` 每次 update 都被循环归档。

## 根因
`update_repo` 的 managed-state 同步链路不幂等：scaffold_v2 新增/改动的文件若没登记在历史 `managed-files.json`，会被判为"本地修改"直接 skipped 并永不再写 hash；同时 `LEGACY_CLEANUP_TARGETS` 错把活跃文件 `.workflow/context/experience/index.md` 当 legacy 归档。

## 影响
- 命令：`harness update`（install 流程不受影响，仅首装安全）
- 文件：`.workflow/context/roles/*.md`、`context/experience/roles/*.md`、`context/experience/risk/known-risks.md` 的模板部分停留在历史版本；`.workflow/context/backup/legacy-cleanup/` 累积 `experience/index.md-N` 垃圾副本
- 目标项目主线数据（`state/`、`flow/`、`artifacts/`）未被主仓库污染；pipx editable 安装本身也正常

## 路由决策
- 回到 `testing`：先补齐两条用例（1. 存量但未登记的 scaffold 文件在 update 后必须到达最新模板；2. 连续两次 update 不得在 `legacy-cleanup` 下产生 `index.md-N`），再进入 executing 修复 `workflow_helpers.py` 的 `_sync_requirement_workflow_managed_files` / `_refresh_managed_state` / `LEGACY_CLEANUP_TARGETS`。

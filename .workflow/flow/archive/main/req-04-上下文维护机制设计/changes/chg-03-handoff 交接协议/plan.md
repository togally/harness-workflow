# 执行计划

## 依赖关系
- **前置依赖**：无硬性依赖，但会被 chg-02、chg-05 引用
- **后置依赖**：为 chg-02、chg-05、chg-06 提供交接协议模板

## 执行步骤

### 步骤 1：分析交接需求
1. 调研新开 agent 时的信息传递需求：
   - 必须传递的核心信息（任务状态、关键决策）
   - 可选传递的辅助信息（会话历史摘要）
   - 需要避免传递的冗余信息
2. 分析现有工作流中的信息载体：
   - `session-memory.md` 的结构和内容
   - `runtime.yaml` 的状态信息
   - 其他相关配置文件

### 步骤 2：设计 handoff.md 模板结构
1. 定义文件基本结构：
   - **标题**：交接协议标识
   - **摘要**：任务概览、交接原因
   - **交接内容**：详细的任务状态和信息
   - **接收方指引**：下一步操作指引
   - **元数据**：时间戳、版本、相关文件
2. 设计必填字段：
   - 任务状态：当前 requirement、stage、进度
   - 关键决策：已做出的重要决策和理由
   - 待处理问题：需要继续处理的问题列表
   - 下一步动作：接收方应该做什么
3. 设计可选字段：
   - 风险提示：已知风险和注意事项
   - 依赖文件：需要传递的文件列表
   - 会话摘要：关键对话片段

### 步骤 3：定义交接内容规范
1. 必须传递的文件列表：
   - `session-memory.md`（完整或摘要）
   - `runtime.yaml`（当前状态）
   - 相关配置文件（如有）
2. 状态信息规范：
   - 如何描述当前 stage 和进度
   - 如何记录已完成的工作
   - 如何标识待处理的问题
3. 会话记忆摘要规范：
   - 如何提取关键对话片段
   - 如何总结重要决策
   - 如何避免信息过载

### 步骤 4：创建接收方指引
1. 加载交接内容的步骤：
   - 如何读取 handoff.md
   - 如何恢复相关文件
   - 如何初始化工作状态
2. 注意事项清单：
   - 未完成的任务项
   - 已知风险和限制
   - 需要特别关注的问题
3. 继续工作的步骤：
   - 如何从交接点继续
   - 如何验证交接完整性
   - 如何报告交接状态

### 步骤 5：创建示例文件
1. 基于实际场景创建完整示例：
   - 选择典型场景（如上下文爆炸需要新开 agent）
   - 填充所有必填字段
   - 展示可选字段的使用
2. 提供简化版示例：
   - 最小必要字段的示例
   - 不同场景的变体示例

### 步骤 6：集成验证
1. 验证与 chg-02 决策树的集成
2. 验证与 chg-05 恢复路径的集成
3. 创建使用指南

## 预期产物
1. `handoff.md` 模板文件
2. `design.md`：包含模板设计说明、使用指南
3. 示例文件（完整版和简化版）

## 验证方法
1. 人工评审模板的完整性和实用性
2. 模拟交接场景验证模板适用性
3. 检查必填字段是否覆盖核心需求
4. 验证与相关变更的集成接口

## 注意事项
1. 保持模板简洁实用，避免过度设计
2. 必填字段要真正必要，可选字段要有明确使用场景
3. 考虑不同场景下的变体需求
4. 确保与工作流其他组件的兼容性

# 执行计划

## 依赖关系
- **前置依赖**：chg-01（监控指标）、chg-02（决策树）
- **后置依赖**：为 chg-05、chg-06 提供工具层支持

## 执行步骤

### 步骤 1：调研 Claude Code 上下文管理工具
1. 详细研究 Claude Code 的上下文管理命令：
   - `/compact`：参数、效果、限制
   - `/clear`：参数、效果、风险
   - `/new`：创建新会话的流程
   - `/help`：相关帮助信息
2. 收集最佳实践和常见错误
3. 记录工具的限制和注意事项

### 步骤 2：创建工具 catalog 条目
1. 创建 `tools/catalog/claude-code-context.md`：
   - **工具定义**：Claude Code 上下文管理能力概述
   - **命令列表**：`/compact`、`/clear`、`/new`、`/help` 的详细说明
   - **用法示例**：各命令的具体使用示例
   - **最佳实践**：何时使用、如何避免问题
   - **与决策树集成**：引用 chg-02 的决策逻辑
2. 确保内容完整、准确、实用

### 步骤 3：更新 stage-tools.md
1. 分析各阶段对上下文管理工具的需求：
   - **planning/executing**：可能需要 `/compact` 继续工作
   - **testing/acceptance**：可能需要 `/clear` 重新测试
   - **regression**：可能需要新开 agent 处理复杂问题
   - **done**：通常不需要上下文管理
2. 更新 `tools/stage-tools.md`：
   - 在各阶段工具白名单中添加上下文管理工具
   - 定义各阶段允许的维护动作
   - 添加使用限制和注意事项

### 步骤 4：补充 selection-guide.md
1. 分析工具选择因素：
   - 上下文负载水平（引用 chg-01 阈值）
   - 当前任务阶段
   - 任务进度和重要性
   - 可用时间和资源
2. 更新 `tools/selection-guide.md`：
   - 增加上下文维护工具的选择指南
   - 提供决策流程图或选择矩阵
   - 引用 chg-02 的决策树作为参考

### 步骤 5：创建经验沉淀模板
1. 在 `context/experience/tool/` 下预留位置：
   - 创建 `claude-code-context.md` 或更新现有文件
   - 设计经验记录模板
   - 预留常见问题和解决方案章节
2. 定义经验沉淀流程：
   - 何时记录上下文管理经验
   - 记录哪些信息
   - 如何分类和检索

### 步骤 6：集成验证
1. 验证与 chg-01 指标的集成
2. 验证与 chg-02 决策树的引用关系
3. 检查工具文档的完整性和一致性

## 预期产物
1. `tools/catalog/claude-code-context.md`：工具 catalog 条目
2. 更新的 `tools/stage-tools.md`：各阶段工具白名单

## 验证方法
1. 人工评审工具文档的准确性和完整性
2. 检查各阶段工具白名单的合理性
3. 验证选择指南的实用性
4. 确认与相关变更的引用关系正确

## 注意事项
1. 工具文档要基于实际使用经验，避免理论化
2. 各阶段的工具限制要符合实际工作需求
3. 选择指南要简单明了，易于使用
4. 确保与工作流其他组件的兼容性
5. 参考经验文件中的教训（如 pipx inject 等）

# 回归简报：bugfix-2 — pytest 3 fail 因测试硬编码旧目录名

## 问题
- `PYTHONPATH=src python3 -m pytest` 跑全量时 `tests/test_chg03_title_contract.py` 与 `tests/test_smoke_req29.py` 共 3 个用例稳定 fail，3 fail 持续跨 req-29/30/31..36 多个生命周期。

## 根因
- 测试 fixture 漂移：`REQ_DIR_NAME` / `REQ_30_DIR_NAME` 与 chg 子目录列表硬编码为 req-29 / req-30 早期 rename 前的旧目录名，没有跟随真实 rename 同步；归档后 active / archive 两路都查不到，断红。

## 影响
- 受影响测试：`test_chg03_title_contract.py::TestReq30SelfCertification` 2 用例 + `test_smoke_req29.py::HumanDocsChecklistTest::test_human_docs_checklist_for_req29` 1 用例
- 受影响命令：`PYTHONPATH=src python3 -m pytest`（全量基线 308 passed → 实际 305 passed + 3 failed）
- 不影响业务代码 / 用户场景，仅测试自身 fixture

## 路由决策
- 真实问题，路由 = executing：直接修 tests/ 硬编码常量为实际归档目录名，并把 req-30 自证测试中"必须存在实施说明.md"的硬断言改为 `pytest.skip`（当前 req-30 rename 后归档下 0 份实施说明.md，是契约 7 fallback 合规存量场景）
- 不动业务代码

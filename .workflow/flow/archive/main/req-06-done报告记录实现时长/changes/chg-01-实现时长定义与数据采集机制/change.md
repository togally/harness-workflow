# Change: chg-01

## Title

实现时长定义与数据采集机制

## Goal

定义"实现时长"的计算口径，评估并选定数据采集方案，确保时长数据稳定可靠。

## Scope

**包含**：
- "实现时长"的计算口径定义（总时长、分阶段时长）
- 数据采集方案评估与选定
- 如需要，更新 `state/requirements/{id}.yaml` 或 `runtime.yaml` 的结构定义
- 降级策略（当部分时间数据缺失时如何展示）

**不包含**：
- 报告模板的具体格式设计（属于 chg-02）
- 实际报告文件的生成（属于 chg-03 验证时）

## Acceptance Criteria

- [ ] 计算口径文档化（总时长 = requirement_review 进入时间 ~ done 结束时间）
- [ ] 数据采集方案已选定并文档化
- [ ] 如涉及 yaml 结构变更，已更新示例/说明
- [ ] 降级策略已定义

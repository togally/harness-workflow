# 实施说明：chg-01 修复 harness regression 命令簇

## 实际做了什么

- 对应 plan.md：新增 `src/harness_workflow/slug.py`，抽出 `slugify_preserve_unicode` util，统一 kebab-case 归一化逻辑（CJK 保留、空格折叠、标点去除、连续编号裁剪），供后续 chg-02 rename/change 路径复用。
- 对应 plan.md：修复 `src/harness_workflow/workflow_helpers.py` 中 `regression_action` 的 `--confirm` 分支：只写回 regression 目录下 meta 的状态，不再消费 `runtime.current_regression`，避免误清空主链路状态。
- 对应 plan.md：改写 `create_regression` 产出目录命名为 `reg-NN-<slug>`（NN 来自新增 `_next_regression_id`），保留 id 前缀便于排序与索引；同时让 `resolve_regression_reference` 支持 `reg-NN-` 前缀匹配，兼容历史引用。
- 对应 plan.md：新增三个内部 helper —— `_next_regression_id`（扫描目录取下一个序号）、`_locate_regression_dir`（按 id 或 slug 定位目录）、`_update_regression_meta_status`（只改 meta 状态不动 runtime），集中状态写入路径，避免散落。
- 对应 plan.md：删除 `.workflow/flow/suggestions/sug-01-regression-confirm-bug.md`，因为该建议已由本 change 正式实施，suggest 池无需再保留。
- 对应 plan.md：新增 `tests/test_regression_helpers.py`，覆盖 slugify 行为、目录命名、引用解析、--confirm 只写 meta 等场景，共 9 个用例全部通过。

## 未做与原因

- chg-02 将复用 `slug.py` 到 `harness rename` 和 `harness change` 命令路径，本 change 为避免越界，未在 rename / change 的代码路径上调用 `slugify_preserve_unicode`，留给 chg-02 统一接入。
- 未改动 scaffold_v2 镜像与角色文件：本 change 纯修 CLI 行为，无对人文档字段或角色契约改动。

- `src/harness_workflow/slug.py`：新增文件，提供 `slugify_preserve_unicode` util。
- `src/harness_workflow/workflow_helpers.py`：`regression_action --confirm` 只写 meta；`create_regression` 产出目录改 `reg-NN-<slug>`；`resolve_regression_reference` 支持 `reg-NN-` 前缀；新增 `_next_regression_id` / `_locate_regression_dir` / `_update_regression_meta_status` 三个 helper。

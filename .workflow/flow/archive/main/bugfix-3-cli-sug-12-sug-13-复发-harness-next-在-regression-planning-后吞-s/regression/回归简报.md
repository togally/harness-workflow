# 回归简报：bugfix-3 CLI sug-12 / sug-13 复发 + archive 选错目标

## 问题
`harness next` 在 regression 后吞 stage（不读 decision.md 路由）；`runtime.yaml` ↔ `state/{requirements,bugfixes}/*.yaml` 的 stage 字段在切换 bugfix / requirement 时不同步；`harness archive` 无参时把列表首个当默认而非 `current_requirement`。

## 根因
- 缺陷 1：`workflow_next`（workflow_helpers.py:6162）完全不读 `runtime.current_regression` 与 `decision.md`，无脑按 sequence + 1 推进。
- 缺陷 2：`enter_workflow`（workflow_helpers.py:6298）只更新 `current_requirement`，不重设 `operation_type`/`operation_target`；`load_requirement_runtime`（workflow_helpers.py:383）懒回填只在为空时触发，残留值不自愈。
- 缺陷 3：`cli.py::main` archive 分支（line 583）调用 `prompt_requirement_selection(..., preselect=args.requirement)`，没把 `current_requirement` 当 preselect 传入。

## 影响
- bugfix-3 — 自身 regression 流；req-36（harness install 同步契约完整性修复）reg-01 / reg-02 复发活证；今日 bugfix-2 enter + 4 次 next + archive 误移走 bugfix-3 工作区的活证。

## 路由决策
- **executing 直接修**：3 个根因点位精确（≤ 4 函数 + 1 模板 + 1 新测试），不需要 requirement_review / planning 重审。修复方案 + 红测已落 `regression/diagnosis.md`。

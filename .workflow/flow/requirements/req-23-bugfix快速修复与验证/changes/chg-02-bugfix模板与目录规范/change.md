# chg-02: bugfix 模板与目录规范

## 变更目标

定义 bugfix 需求的专属产物结构和 `bugfix.md` 模板，确保 bugfix 流程有统一的三合一天然产物。

## 变更范围

- 新建 `src/harness_workflow/assets/templates/bugfix.md`（或等效的 scaffold 模板位置）
- 确认 `.workflow/flow/bugfixes/` 目录规范在文档中落地

## 验收标准

- [ ] `bugfix.md` 模板包含：问题描述、根因分析、修复范围、修复方案、验证标准
- [ ] 模板文件已放入项目模板目录
- [ ] 文档中明确 bugfix 目录结构与标准需求目录的区别

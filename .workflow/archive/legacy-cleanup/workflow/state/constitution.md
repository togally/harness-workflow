# Constitution

## 核心原则

1. **生产者和评估者必须独立** — 执行 agent 和验收 agent 不能是同一个实例
2. **经验驱动改进** — 每个阶段结束都应检查是否有可沉淀的经验
3. **薄入口，按需加载** — 不一次性读取所有上下文，通过索引按需加载
4. **状态驱动恢复** — 任何中断都应能通过状态文件恢复，不依赖对话历史
5. **失败路径必须记录** — 走过的失败路径要写进 session-memory，不重复踩坑
6. **诊断优先于修复** — 发现问题先确认是真实问题，再开始修复
7. **上下文爆满时交接，不压缩** — 写 handoff.md，新起 agent 接管

## 协作偏好

- 直接给出判断和建议，不要过度询问
- 发现设计问题时主动提出，不等用户发现后再说
- 经验沉淀要具体，不要泛化成废话

## 不可违背的约定

- 任何工作必须先读 runtime.yaml 确认当前状态
- 节点任务不由主 agent 直接执行
- 没有 current_requirement 不进入任何工作阶段
